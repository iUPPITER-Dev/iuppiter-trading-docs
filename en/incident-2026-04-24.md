# Incident Retrospective: 2026-04-24 IUPUSDT 11.2× Swing → Asymmetric Defense System

> **Summary**: An 11.2× price swing caused not by an external attack but by **internal failure (ghost bot + dual-run + Walk-Up bug)**. After diagnosis, a 5-layer asymmetric defense system was built and verified to block the incident at 100% in backtest and preserve 100% of capital across 14h of live observation.

---

## 1. Incident Summary

**Window**: 2026-04-24 KST 07:00 ~ 13:00  
**Symptoms**: IUPUSDT price moved +320% spike then -95% drop within 60min, total swing **11.2×**  
**Capital impact**: 27 recoverable ghost orders ($135) + some USDT loss  
**Initial hypothesis**: External attacker  
**Actual cause**: **Internal system failure**

---

## 2. Root Cause (3 Concurrent Failures)

### 2.1 Ghost Bot (BotRehydrator race)
**Issue**: Bot instance was spawned twice (rehydrator started the same bot twice)  
**Result**: Two instances collided on cancel/place, leaving some orders in a "ghost" state  
**Root cause**: BotRehydrator lacked a distributed lock → concurrency race condition

### 2.2 Walk-Up Direction Bug
**Issue**: Walk-Up Engine's target reverse branch triggered runaway **sell self-cross**  
**Result**: Our bot pushed price down → mid dropped → external detector panicked into selling

### 2.3 External Detector Bot Panic-Buy
**Situation**: When mid showed an abnormal pattern (our self-cross runaway), external bots tried spike trades  
**Result**: External panic-buy → spike formed → sell wall built → 95% drop

→ **Not an external attack — our race triggered market detector bots.**

---

## 3. Diagnosis → Asymmetric Defense Architecture

5-layer redundancy:

| Layer | Role | Module |
|---|---|---|
| **1. Mode FSM (5-state)** | Auto transitions NORMAL → ENGAGED_BUY/SELL → DEFENSIVE → CRASH → RECOVERY + 5min cool-down | `mode-fsm.ts` |
| **2. Asymmetric Ladder** | Per-mode bid/ask side spread/size multipliers | `asymmetric-ladder.ts` |
| **3. Circuit Breakers (5)** | NetTaker / PriceMove / StopLoss / InventorySkew / BookSanity — auto-trip on threshold breach | `circuit-breakers/` |
| **4. Symbol Adaptive Profile** | Per-symbol volatility profile fan-out, threshold auto-calibration | `symbol-adaptive-profile-store.ts` |
| **5. Tick History Aggregator** | 1-min mode/breaker/flow rollup → DB → audit/retrospective | `tick-history-aggregator.ts` |

**SaaS boundary principle**: per-bot / per-user only. **No global kill switch** (multi-tenant safety).

**Observability**:
- Standard tier: ModeBadge + EmergencyStop button
- Pro tier: DefenseStatusPanel + real-time breaker telemetry

## 6. Side Discoveries + Code Fixes

Two separate wiring bugs surfaced during live observation:

### 6.1 Walk-Up `INIT` Stuck Forever
- **Symptom**: Walk-Up state stayed `INIT` for 14h, self-cross fill_count = 0
- **Cause**: Launcher passed only `priceBand`, omitted `targetPrice` → `WalkUpPolicy.active === false` → `observe()` returned at the gate
- **Fix**: `feb1d61` launcher passes `targetPrice` directly + volume-mm-bot derives priceBand center as fallback

### 6.2 tick_history fill_count Placeholder
- **Symptom**: Bot self-crossed normally but metrics aggregated as 0 (8 best-pair fills, contradicting prod log)
- **Cause**: P4-3 work left `taker_count: 0  // TODO: wire real count` placeholder unfinished
- **Fix**: `f4a372c` wired to `lastBestPairFilled === true` signal (1 best-pair cycle = fill_count 2)

## 7. Lessons

1. **Don't assume external attack** — internal failures are more common, start with diagnosis
2. **Distributed locks are required** — concurrency code like BotRehydrator must be guarded with Redis NX or similar
3. **Multi-layer redundancy** — PriceMove + NetTaker + InventorySkew run together, fallback when one fails
4. **Multi-tenant boundary** — global kill switch lets one user's incident kill another user's bot, forbidden
5. **Don't trust metrics alone** — placeholder TODO is possible, cross-check with prod log + balance + other signals
6. **Market flow analysis** — measure 4 indicators before climb attempt ([market-analysis.md](./market-analysis.md))
