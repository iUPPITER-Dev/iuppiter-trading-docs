# ボット種別概要 + 選択ガイド

> **対象**: どのボットをどの状況で使うか決める運用者。
> **要点**: 3 種類の専門化されたボット — 目的が異なればボットも異なる。

---

## 1. 一目で

| ボット | 一文の目的 | 主な収益源 | 適した利用者 |
|---|---|---|---|
| **Volume-MM** | 人為的な取引量 + 価格 climb | 手数料インセンティブ / 市場活性度 | トークン発行者、上場基準充足 |
| **Role-MM** | 真のマーケットメイキング (スプレッド差益) | 両側の bid/ask スプレッド | 専門マーケットメイカー |
| **Arbitrage** | 取引所間価格差益 | 取引所間の価格差 | 裁定取引運用者 |

**詳細な運用 setting は各ボット別ガイドを参照** (現在 Volume-MM のみ docs 提供、Role-MM/Arbitrage は今後追加予定)。

## 2. Volume-MM ボット

### 何をするか
- 同じ user の二つの sub-account (bid wallet / ask wallet) 間の自己クロスで取引量生成
- オプション: 価格を段階的に上昇 (Walk-Up Engine)
- 自動防御システム (Asymmetric Defense): 異常市場検知時 ladder 封鎖

### いつ使うか
- 新規トークン発行後の取引量不足 — 取引所の上場/再上場基準を満たす必要
- 特定価格帯の維持 — 板の活性度維持
- 段階的な価格上昇 — 強気の流れで自然な climb

### 適した市場
✅ 安定的な流れ (変動性 < 5% 日次)  
✅ 買い / 売り両方向のバランス (24h flow score -0.3 ~ +0.3)  
✅ 外部トレーダーが一定数存在 (最低 10 trades/h)

### 不適な市場
❌ 強い売り優勢 (24h flow score < -0.5) — climb 試行時 USDT 流出
❌ 厚い売り壁 + 薄い買い厚み — 一方的な吸収リスク
❌ 極低流動性 (< 5 trades/h) — fill 自体が困難

### 資本ガイド
- 最小 USDT $300 + base asset 一定量 (資産数量は価格に対し minNotional 通過)
- 推奨 USDT $600+ × 2-3 wallet — 非対称防御が自由に動作する余裕

### 運用モード (選択)
1. **自己クロス only (安全 1 順位)**: walkUp 無効、価格変動試行なしで取引量のみ — [safe-recipe.md](./safe-recipe.md)
2. **Walk-Up climb (市場回復期)**: walkUp.enabled=true、target 価格段階的上昇 — 市場フロー分析後に決定 ([market-analysis.md](./market-analysis.md))

## 3. Role-MM ボット

### 何をするか
- 3 つの sub-account: bid (USDT 買い側) / ask (coin 売り側) / settle (再バランシング清算)
- **両側の板に同時に limit order 発行** — 片側約定時にスプレッド差益確保
- 市場の変動性に応じてスプレッド自動調整 (Avellaneda-Stoikov アルゴリズム)
- 片側資産枯渇時、settle 計座を経由して自動再バランシング

### いつ使うか
- 安定的なマーケットメイキングで **継続的なスプレッド収益** 追求
- トークンの自然な板維持 (両側深さの提供)
- ヘッジ統合 — ある取引所の現物 MM + 別の取引所のヘッジ (XEMM)
- 大量ポジションを市場価格衝撃なしで徐々に整理 (TWAP)

### 適した市場
✅ 中堅トークン (適度な取引量、安定的 spread)  
✅ 低〜中変動性 (Avellaneda モデルが効果的)  
✅ 取引所自体が sub-account 自動移管をサポート (Binance / OKX / KuCoin 等)

### 不適な市場
❌ 急騰落相場 (Avellaneda の動的スプレッドが逆マージン可能)
❌ 極低流動性 (在庫枯渇頻繁)
❌ 超高手数料取引所 (spread < 手数料 で損失)

### 資本ガイド
- 最小 USDT $500 × 3 wallet (bid + ask + settle)
- 推奨 USDT $2000+ × 3 — 在庫変動吸収 + 持続運用
- coin 保有量は日平均取引量 × 1-2 日分

### Volume-MM との違い
- Volume-MM: 自己クロス (同じ user の二つの wallet 同士) → 取引量生成、収益なし
- Role-MM: 実際の外部トレーダーと売買 → スプレッド差益、真の市場形成

## 4. Arbitrage ボット

### 何をするか
- 二つの取引所 (例: MEXC + Binance) の板をリアルタイム比較
- 価格差が手数料 + 閾値より大きければ自動売買:
  - 安い取引所で買い + 高い取引所で売り (同時発注)
- 部分約定リスク管理 (リトライ + 不均衡ガード)

### いつ使うか
- 新規トークン上場直後 — 二つの取引所間の価格不一致が大きい
- 遊休資本効率化 — 理論的にノーリスク (両側同時約定時)
- 両取引所間の資本均衡自動調整

### 適した市場
✅ 新規上場トークン (価格発見段階、差益 0.5~5% 可能)  
✅ 低〜中流動性ペア (メジャーコインは差益ほぼゼロ)  
✅ 二つの取引所が安定的なレイテンシ

### 不適な市場
❌ メジャーコイン (BTC, ETH 等) — 差益 < 0.05% で手数料を賄えない
❌ 高ネットワーク遅延 — 片側のみ約定で位置リスク
❌ 手数料 > 0.5% 取引所 — 純益マイナス

### 資本ガイド
- 最小 USDT $1000 × 2 取引所
- 推奨 USDT $5000+ × 2 — 大きな機会捕捉 + 部分約定吸収

### リスク要因
- **部分約定**: 片側のみ約定 → 一方的なポジション → 価格逆転で損失
- **取引所遅延**: WS データ古い → 消えた機会を追撃 → 損失
- **手数料**: 両側手数料合計 > 差益 で純損失

## 5. ボット選択決定マトリックス

| 状況 / 目的 | 推奨ボット |
|---|---|
| 新規トークン、上場基準取引量必要 | **Volume-MM** (自己クロス only) |
| 新規トークン、価格発見 + 二つの取引所差益 | **Arbitrage** |
| 安定トークン、継続的スプレッド収益 | **Role-MM** |
| トークン価格段階的上昇 + 取引量同時 | **Volume-MM** (Walk-Up + 市場分析) |
| 大きなポジション徐々に整理 (市場衝撃なし) | **Role-MM** (TWAP executor) |
| 両取引所ヘッジ (現物+先物または二つの取引所) | **Role-MM** (XEMM) |
| 資本 < $1000 | **Volume-MM** (最も資本効率良い) |

## 6. 共通安全原則 (全ボット)

1. **開始前に市場分析** — [market-analysis.md](./market-analysis.md) の 4 指標測定
2. **資本は保守的に** — 最初は少額で検証、安定確認後に増額
3. **継続的モニタリング** — 最初の 6h は 1h 単位の点検推奨
4. **アラーム基準設定** — USDT 開始対比 -2% 到達時 stop 推奨 (保守)
5. **防御システム信頼** — Circuit Breaker / Defense Mode が自動動作
