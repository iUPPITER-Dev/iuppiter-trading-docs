# 봇 종류별 기능 개요 + 선택 가이드

> **대상**: 어떤 봇을 어떤 상황에 써야 할지 결정해야 하는 운영자.
> **요지**: 3 종류의 전문화된 봇 — 목적이 다르면 봇도 달라진다.

---

## 1. 한눈에 보기

| 봇 | 한 문장 목적 | 주 수익원 | 적합한 사용자 |
|---|---|---|---|
| **Volume-MM** | 인위적 거래량 + 가격 climb | 수수료 인센티브 / 시장 활성도 | 토큰 발행자, 상장 기준 충족 |
| **Role-MM** | 진정한 마켓메이킹 (스프레드 차익) | 양쪽 호가 스프레드 | 전문 마켓메이커 |
| **Arbitrage** | 두 거래소 가격 편차 차익 | 거래소 간 가격 차이 | 차익거래 운영자 |

**자세한 운영 setting 은 각 봇 별 가이드 참조** (현재 Volume-MM 만 docs 제공, Role-MM/Arbitrage 향후 추가 예정).

## 2. Volume-MM 봇

### 무엇을 하나
- 같은 user 의 두 sub-account (bid wallet / ask wallet) 사이 자전거래로 거래량 생성
- 옵션: 가격을 점진적으로 상승 (Walk-Up Engine)
- 자동 방어 시스템 (Asymmetric Defense): 비정상 시장 감지 시 ladder 봉쇄

### 언제 쓰나
- 신규 토큰 발행 후 거래량 부족 — 거래소 상장/재상장 기준 충족 필요
- 특정 가격대 유지 — 호가창 활성도 유지
- 점진적 가격 상승 — bullish 흐름에서 자연스러운 climb

### 적합한 시장
✅ 안정적 흐름 (변동성 < 5% 일간)  
✅ 매수 / 매도 양방향 균형 (24h flow score -0.3 ~ +0.3)  
✅ 외부 거래자 일정량 존재 (최소 10 trades/h)

### 부적합한 시장
❌ 강한 매도 우세 (24h flow score < -0.5) — climb 시도 시 USDT 출혈
❌ 두꺼운 매도벽 + 얇은 매수 매물대 — 일방적 흡수 위험
❌ 극저 유동성 (< 5 trades/h) — fill 자체가 어려움

### 자본 가이드
- 최소 USDT $300 + base asset 일정량 (자산 수량은 가격 대비 minNotional 통과)
- 권장 USDT $600+ × 2-3 wallet — 비대칭 방어가 자유롭게 작동할 여유

### 운영 모드 (선택)
1. **자전거래 only (안전 1순위)**: walkUp 비활성, 가격 변동 시도 없이 거래량만 — [safe-recipe.md](./safe-recipe.md)
2. **Walk-Up climb (시장 회복기)**: walkUp.enabled=true, target 가격 점진 상승 — 시장 흐름 분석 후 결정 ([market-analysis.md](./market-analysis.md))

## 3. Role-MM 봇

### 무엇을 하나
- 3 개의 sub-account: bid (USDT 매수 측) / ask (coin 매도 측) / settle (재밸런싱 청산)
- **양쪽 호가에 동시에 limit order 발행** — 한쪽 체결 시 스프레드 차익 확보
- 시장 변동성에 따라 스프레드 자동 조정 (Avellaneda-Stoikov 알고리즘)
- 한쪽 자산 고갈 시 settle 계정으로 자동 재밸런싱

### 언제 쓰나
- 안정적 마켓메이킹으로 **지속적 스프레드 수익** 추구
- 토큰의 자연스러운 호가창 유지 (양쪽 깊이 제공)
- 헷지 통합 — 한 거래소 현물 마켓메이킹 + 다른 거래소 헷지 (XEMM)
- 대량 포지션을 시장가 충격 없이 천천히 정리 (TWAP)

### 적합한 시장
✅ 중체급 토큰 (적당한 거래량, 안정적 spread)  
✅ 낮은~중간 변동성 (Avellaneda 모델이 효과적)  
✅ 거래소 자체가 sub-account 자동 이체 지원 (Binance / OKX / KuCoin 등)

### 부적합한 시장
❌ 급등락 장세 (Avellaneda 의 동적 스프레드 가 역마진 가능)
❌ 극저 유동성 (재고 고갈 빈번)
❌ 초고 수수료 거래소 (스프레드 < 수수료 시 손실)

### 자본 가이드
- 최소 USDT $500 × 3 wallet (bid + ask + settle)
- 권장 USDT $2000+ × 3 — 재고 편차 흡수 + 지속 운영
- coin 보유량은 일평균 거래량 × 1-2 일치

### Volume-MM 과의 차이
- Volume-MM: 자전거래 (같은 user 의 두 wallet 끼리) → 거래량 생성, 수익 X
- Role-MM: 실제 외부 거래자와 매매 → 스프레드 차익, 진정한 시장 형성

## 4. Arbitrage 봇

### 무엇을 하나
- 두 거래소 (예: MEXC + Binance) 의 호가창을 실시간 비교
- 가격 차이가 수수료 + 임계값 보다 크면 자동 매매:
  - 저가 거래소 매수 + 고가 거래소 매도 (동시 발주)
- 부분 체결 위험 관리 (재시도 + 불균형 가드)

### 언제 쓰나
- 신규 토큰 상장 직후 — 두 거래소 간 가격 불일치 큼
- 유휴 자본 효율화 — 이론적으로 무위험 (양쪽 동시 체결 시)
- 양 거래소 자본 균형 자동 조절

### 적합한 시장
✅ 신규 상장 토큰 (가격 발견 단계, 차익 0.5~5% 가능)  
✅ 저~중간 유동성 페어 (메이저 코인은 차익 거의 없음)  
✅ 두 거래소 모두 안정적 latency

### 부적합한 시장
❌ 메이저 코인 (BTC, ETH 등) — 차익 < 0.05% 라 수수료 못 따라옴
❌ 높은 네트워크 지연 — 한쪽만 체결되어 위치험 발생
❌ 수수료 > 0.5% 거래소 — 순이익 음수

### 자본 가이드
- 최소 USDT $1000 × 2 거래소
- 권장 USDT $5000+ × 2 — 큰 기회 포착 + 부분 체결 흡수

### 위험 요소
- **부분 체결**: 한쪽만 체결 → 일방적 포지션 → 가격 reversal 시 손실
- **거래소 지연**: WS 데이터 stale → 사라진 기회 추격 → 손실
- **수수료**: 양쪽 거래 수수료 합 > 차익 시 음수

## 5. 봇 선택 결정 매트릭스

| 상황 / 목적 | 추천 봇 |
|---|---|
| 신규 토큰, 상장 기준 거래량 필요 | **Volume-MM** (자전거래 only) |
| 신규 토큰, 가격 발견 + 두 거래소 차익 | **Arbitrage** |
| 안정 토큰, 지속 스프레드 수익 | **Role-MM** |
| 토큰 가격 점진 상승 + 거래량 동시 | **Volume-MM** (Walk-Up + 시장 분석) |
| 큰 포지션 천천히 정리 (시장 충격 X) | **Role-MM** (TWAP executor) |
| 양 거래소 헷지 (현물+선물 또는 두 거래소) | **Role-MM** (XEMM) |
| 자본 < $1000 | **Volume-MM** (가장 자본 효율) |

## 6. 공통 안전 원칙 (모든 봇)

1. **시작 전 시장 분석** — [market-analysis.md](./market-analysis.md) 의 4 지표 측정
2. **자본은 보수적으로** — 처음엔 작은 금액으로 검증, 안정 확인 후 증액
3. **모니터링 지속** — 첫 6h 는 1h 단위 점검 권장
4. **알람 기준 설정** — USDT 시작 대비 -2% 도달 시 stop 권장 (보수)
5. **방어 시스템 신뢰** — Circuit Breaker / Defense Mode 가 자동 작동
