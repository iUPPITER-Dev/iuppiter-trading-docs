# 机器人类型概览 + 选择指南

> **对象**: 决定哪个机器人适合自己情况的运营者。
> **要点**: 3 种专业化的机器人 — 目的不同,机器人也不同。

---

## 1. 一目了然

| 机器人 | 一句话目的 | 主要收益源 | 适合的用户 |
|---|---|---|---|
| **Volume-MM** | 人为的交易量 + 价格 climb | 手续费激励 / 市场活跃度 | 代币发行者,上市标准达成 |
| **Role-MM** | 真正的做市 (价差差益) | 双边 bid/ask 价差 | 专业做市商 |
| **Arbitrage** | 两家交易所价格差益 | 交易所间价格差 | 套利运营者 |

**详细运营 setting 参考各机器人专属指南** (目前仅 Volume-MM 有 docs 提供,Role-MM/Arbitrage 后续增加)。

---

## 2. Volume-MM 机器人

### 做什么
- 同一 user 的两个 sub-account (bid wallet / ask wallet) 之间自交易生成交易量
- 可选: 逐步推升价格 (Walk-Up Engine)
- 自动防御系统 (Asymmetric Defense): 检测到异常市场时封锁 ladder

### 何时使用
- 新代币发行后交易量不足 — 需达成交易所的上市/复市标准
- 维持特定价格区间 — 保持订单簿活跃
- 渐进式价格上涨 — 在牛市流向中自然 climb

### 适合的市场
✅ 稳定流向 (波动性 < 5% 日内)  
✅ 买/卖双向平衡 (24h flow score -0.3 ~ +0.3)  
✅ 一定数量的外部交易者 (至少 10 trades/h)

### 不适合的市场
❌ 强烈卖压 (24h flow score < -0.5) — climb 尝试 USDT 流失
❌ 厚卖墙 + 薄买方流动性 — 单方向吸收风险
❌ 极低流动性 (< 5 trades/h) — fill 本身困难

### 资本指南
- 最少 USDT $300 + base asset 一定量 (资产数量需对应价格通过 minNotional)
- 推荐 USDT $600+ × 2-3 wallet — 给非对称防御充分运作的余地

### 运营模式 (选择)
1. **自交易 only (安全第一)**: walkUp 禁用,不尝试价格变动只生成交易量 — [safe-recipe.md](./safe-recipe.md)
2. **Walk-Up climb (市场恢复期)**: walkUp.enabled=true,target 价格逐步上升 — 市场流向分析后决定 ([market-analysis.md](./market-analysis.md))

## 3. Role-MM 机器人

### 做什么
- 3 个 sub-account: bid (USDT 买侧) / ask (coin 卖侧) / settle (再平衡清算)
- **同时在双边订单簿挂 limit order** — 任一侧成交时锁定价差差益
- 根据市场波动性自动调整价差 (Avellaneda-Stoikov 算法)
- 单侧资产耗尽时通过 settle 账户自动再平衡

### 何时使用
- 稳定做市追求 **持续价差收益**
- 提供代币自然订单簿 (双边深度)
- 对冲集成 — 一个交易所现货 MM + 另一个交易所对冲 (XEMM)
- 大量仓位无市场冲击地慢慢清理 (TWAP)

### 适合的市场
✅ 中型代币 (适度交易量、稳定 spread)  
✅ 低到中等波动性 (Avellaneda 模型最有效)  
✅ 交易所支持 sub-account 自动转账 (Binance / OKX / KuCoin 等)

### 不适合的市场
❌ 急涨急跌行情 (Avellaneda 的动态价差可能负利润)
❌ 极低流动性 (库存频繁耗尽)
❌ 超高手续费交易所 (价差 < 手续费 → 损失)

### 资本指南
- 最少 USDT $500 × 3 wallet (bid + ask + settle)
- 推荐 USDT $2000+ × 3 — 吸收库存波动 + 持续运营
- coin 持有量为日均交易量 × 1-2 日

### 与 Volume-MM 的差异
- Volume-MM: 自交易 (同一 user 的两个 wallet 之间) → 生成交易量,无收益
- Role-MM: 与真实外部交易者交易 → 价差差益,真正的市场形成

## 4. Arbitrage 机器人

### 做什么
- 实时比较两个交易所 (例如 MEXC + Binance) 的订单簿
- 价格差大于手续费 + 阈值时自动交易:
  - 低价交易所买入 + 高价交易所卖出 (同时发单)
- 部分成交风险管理 (重试 + 不平衡守卫)

### 何时使用
- 新代币上市直后 — 两交易所间价格不一致显著
- 闲置资本效率化 — 理论上无风险 (双侧同时成交时)
- 两交易所间资本均衡自动调节

### 适合的市场
✅ 新上市代币 (价格发现阶段,0.5~5% 差益可能)  
✅ 低到中流动性对 (主流币几乎没差益)  
✅ 两交易所 latency 稳定

### 不适合的市场
❌ 主流币 (BTC, ETH 等) — 差益 < 0.05% 不能覆盖手续费
❌ 高网络延迟 — 仅一侧成交 → 仓位风险
❌ 手续费 > 0.5% 交易所 — 净利润为负

### 资本指南
- 最少 USDT $1000 × 2 交易所
- 推荐 USDT $5000+ × 2 — 捕捉大机会 + 吸收部分成交

### 风险要素
- **部分成交**: 仅一侧成交 → 单方向仓位 → 价格反转时损失
- **交易所延迟**: WS 数据陈旧 → 追逐已消失的机会 → 损失
- **手续费**: 双侧手续费合计 > 差益 → 净负

## 5. 机器人选择决策矩阵

| 状况 / 目标 | 推荐机器人 |
|---|---|
| 新代币,需上市标准交易量 | **Volume-MM** (自交易 only) |
| 新代币,价格发现 + 两交易所差益 | **Arbitrage** |
| 稳定代币,持续价差收益 | **Role-MM** |
| 代币价格逐步上涨 + 交易量同时 | **Volume-MM** (Walk-Up + 市场分析) |
| 大仓位慢慢清理 (无市场冲击) | **Role-MM** (TWAP executor) |
| 两交易所对冲 (现货+期货或两交易所) | **Role-MM** (XEMM) |
| 资本 < $1000 | **Volume-MM** (资本效率最高) |

## 6. 通用安全原则 (所有机器人)

1. **启动前进行市场分析** — 测量 [market-analysis.md](./market-analysis.md) 的 4 指标
2. **资本保守** — 起始小额验证,确认稳定后增加
3. **持续监控** — 首 6h 推荐 1h 单位检查
4. **设置警报阈值** — USDT 起始相比 -2% 达到时 stop 推荐 (保守)
5. **信任防御系统** — Circuit Breaker / Defense Mode 自动启动
